## Term status

Architecture metaphor.

## Meaning

Shared Context Layer describes a proposed shared or distributed layer for exchanging the stated resource across systems.

## Boundary

The label does not define an interoperability protocol, trust model or data contract. Those must be specified independently.

## How it is used

Shared Context Layer is used when multiple systems need access to the same [[Governed Context|governed context]] sources and rules. It is the term to use when you want a shared service boundary rather than each agent inventing its own retrieval stack.
