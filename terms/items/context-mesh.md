## Term status

Architecture metaphor.

## Meaning

Context Mesh describes a proposed shared or distributed layer for exchanging the stated resource across systems.

## Boundary

The label does not define an interoperability protocol, trust model or data contract. Those must be specified independently.

## How it is used

Context Mesh is used when teams need to reason about the information presented to a model at decision time. It should result in explicit source selection, provenance, freshness, permissions and token-budget behaviour.
