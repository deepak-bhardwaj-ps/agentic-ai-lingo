An agent [[Registry|registry]] is a directory of agents. It stores basic facts about each agent, such as its name, what it does, who owns it, and how other systems can reach it.

In practice, it works like a contact list for software agents. Instead of hunting through code, chats, or random documents, a system can look up one place to find an agent and read its details.

This matters because teams can end up with many agents quickly. A [[Registry|registry]] helps people keep track of them, search for them, and see which ones are meant to be used.

It is not the same as trust. Being listed does not mean an agent is safe, correct, or approved. It also does not give an agent permission to act. A [[Registry|registry]] is a record, not a security check.
