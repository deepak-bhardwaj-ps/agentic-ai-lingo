## Term status

Emerging interoperability/architecture label.

## Meaning

Agent Architect describes the person who shapes an agent system before implementation: which agents exist, how they connect, what each may access, and where control really sits.

## Boundary

The label does not make the work a protocol. Interoperability only exists if there is a public wire format, lifecycle rules, compatibility constraints, and a security model.

## How it is used

The term shows up when one person owns the shape of the whole system rather than isolated demos. The useful work is in interfaces, [[Guardrails|guardrails]], escalation paths, and operational fit.

[Model Context Protocol specification](https://modelcontextprotocol.io/specification/2025-06-18) is the relevant reference point. For coined labels, it supports the underlying concept, not the claim that the title is standardised.
