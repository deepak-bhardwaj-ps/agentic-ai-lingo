Long-Horizon Workflow means a task that has to stay on track over many steps, not just one quick action.

In practice, this means the work may need planning, checkpoints, retries, and a way to remember what happened earlier. If something goes wrong, the workflow should be able to pick up again without losing the goal.

This matters because long tasks are where mistakes tend to build up. A small error early on can spoil later steps, so the process needs regular checking and recovery.

It is not the same as being fully autonomous. It is also not just a task that takes a long time. The key idea is that the workflow must remain coherent while it unfolds.
