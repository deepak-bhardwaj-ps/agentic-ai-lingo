Agent FOMO means feeling pressure to add an AI agent because other people are using them, not because the task really needs one.

In practice, it means a team may be chasing a trend before checking the problem, the risks, and whether a simpler tool would work better. If a task is mostly fixed, predictable, and easy to automate, an agent can add extra cost, mistakes, and confusion instead of helping.

The term matters because weak technology choices often start with pressure, not evidence. Good decisions ask what the user needs, what the current process already does, how success will be measured, and whether the system can be controlled safely.

It is not a formal technical term. It does not mean agents are always bad. It means they were chosen too quickly, or for the wrong reason. It also is not the same as good design or automation: sometimes an agent is the right tool, but only after the problem has been checked properly.
